# Reglas Git

Nunca crear commits gigantes.

Cada commit debe representar una unidad lógica.

Ejemplos:

feat(i18n): add translation infrastructure

fix(security): upgrade vulnerable dependencies

refactor(styles): centralize design tokens

test(auth): add login coverage

chore(deps): update safe minor versions

docs(traceability): add maintenance log

Máximo recomendado:

1 propósito por commit.
